# Sov Upgrade Map

Interactive EVE Online sovereignty map.

## Frontend

Requires Node.js 20 or newer.

```bash
cd frontend
npm install
npm run generate:map-data
npm run generate:sde-i18n
npm run dev
```

The frontend uses Rsbuild and React. Static starmap data is generated from the official
EVE Online SDE JSONL archive into `frontend/src/data/map.generated.ts`; the browser no
longer fetches `/api/public/map/dataset`.
Type IDs returned by the backend are also resolved in the browser from bundled SDE
locale files generated by `npm run generate:sde-i18n`.

During development, Rsbuild proxies `/api` to `http://localhost:8080`.

## Backend

```bash
cd backend
cargo run
```

The backend is Rust/Axum and exposes:

```text
GET /healthz
GET /api/auth/eve/authorize
GET /api/public/sovereignty/status
```

It polls public ESI directly:

- `GET /sovereignty/systems`, every 5 minutes
- `GET /skyhooks/raidable`, every 15 minutes

Authenticated sovereignty hub details are read from:

```text
GET /corporations/{corporation_id}/structures/sovereignty-hubs/{sovereignty_hub_id}
```

Provide tokens in `backend/tokens.json` or set `SOV_TOKEN_FILE` to another JSON file.
See `backend/tokens.example.json` for the expected shape. Tokens with refresh metadata
are refreshed before hub detail polling and persisted back to the token file when ESI
rotates token values. Tokens used for hub details need the
`esi-structures.read_corporation.v1` scope.

To authorize a character for EVE Online scopes, configure `EVE_SSO_CLIENT_ID` and
`EVE_SSO_REDIRECT_URI`, then open:

```text
GET /api/auth/eve/authorize?state={csrf-state}
```

The endpoint redirects to the EVE SSO authorization-code flow using
`EVE_SSO_SCOPES`. A one-off `scope` or `scopes` query parameter can override the
configured scopes for that request, using comma or space separated values.

Useful environment variables:

| Name | Default | Purpose |
| --- | --- | --- |
| `LISTEN_ADDR` | `127.0.0.1:8080` | Backend listen address |
| `ESI_BASE_URL` | `https://esi.evetech.net` | ESI API base URL |
| `ESI_USER_AGENT` | `sov-upgrade-map/0.1 (https://esi.evetech.net)` | User-Agent sent by the backend HTTP client |
| `SOV_TOKEN_FILE` | `tokens.json` | Auth token store for sovereignty hub detail calls |
| `EVE_SSO_CLIENT_ID` | unset | EVE developer application client ID used by `/api/auth/eve/authorize` |
| `EVE_SSO_REDIRECT_URI` | unset | Registered EVE SSO callback URL used by `/api/auth/eve/authorize` |
| `EVE_SSO_SCOPES` | `esi-structures.read_corporation.v1` | Comma or space separated scopes requested by default |
| `EVE_SSO_AUTHORIZE_URL` | `https://login.eveonline.com/v2/oauth/authorize` | EVE SSO authorization endpoint |
| `RUST_LOG` | `sov_upgrade_map_backend=info,tower_http=info` | Logging filter |
